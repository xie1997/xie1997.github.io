K8S 配置身份认证

云安全
技术文章
发布日期:   2022-11-03
更新日期:   2023-03-07
文章字数:   1.4k
阅读时长:   6 分
阅读次数:  

User

以下演示新建用户，并且赋予用户在指定命名空间和集群的相关权限。指定–kubeconfig参数后，所有的配置都会写到对应的文件中。如果不指定–kubeconfig参数那么就会将配置文件写到~/.kube/config这个文件中。

创建用户对指定namespace具有权限

以下演示新建用户test，并且赋予test用户在test命名空间中对pod拥有指定权限。以下不单独生成配置文件，将配置写入~/.kube/config文件。

生成证书

#生成私钥test.key
openssl genrsa -out test.key 2048
#用该私钥创建一个CSR证书签名请求文件test.csr
openssl req -new -key test.key -subj "/CN=test/O=goup1/O=goup2" -out test.csr
#用CA的私钥对该CSR请求签发一个证书test.crt
openssl x509 -req -in test.csr -CA /etc/kubernetes/pki/ca.crt -CAkey /etc/kubernetes/pki/ca.key -CAcreateserial -out test.crt -days 365

设置用户

#设置用户，使用客户端的证书和客户端密钥，将证书数据嵌入到配置文件中。
kubectl config set-credentials test --client-certificate=./test.crt --client-key=./test.key --embed-certs=true
#删除用户
kubectl config unset users.test

设置上下文信息

在kubeconfig中，context中将访问一个集群的参数进行分组。访问这个context名称就是访问这个参数组。context就是一组信息的别名。context将集群、命名空间、用户进行分组。每个context对应一个环境信息。

每个context都有3个参数：

  • cluster （集群）
  • namespace（命名空间）
  • user（用户）

默认情况下，kubectl命令从current context中来获取参数，然后与集群进行通讯。

#设置上下文
kubectl config set-context test@kubernetes --cluster=kubernetes --user=test --namespace=test
#查看配置中所有的context
kubectl config get-contexts

#查看当前的context
kubectl config current-context
#切换上下文
kubectl config use-context test@kubernetes
#删除context
kubectl config delete-context test@kubernetes

创建角色role

接着创建test-role角色，作用在test命名空间，对pod拥有指定的权限。

kubectl apply -f create-role.yml

create-role.yml文件如下。

kind: Role
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  namespace: test
  name: test-role
rules:
- apiGroups: [""]
  resources: ["pods"]
  verbs: ["get", "list", "watch", "create", "update", "patch", "delete"]

角色绑定

将刚刚创建的test-role角色与test用户进行绑定，作用在test命名空间。

kubectl create rolebinding test-rolebinding -n test --role=test-role --user=test

访问

最后切换上下文到刚刚创建的test用户，可以看到只对test命名空间的pod具有权限。

#查看上下文
kubectl config get-contexts
#切换上下文
kubectl config use-context test@kubernetes
#查看pod
kubectl get pods -n test

创建用户对集群具有权限

以下演示新建用户test2，并且赋予test2用户在集群中对pod拥有指定权限。以下单独生成配置文件test2-config。

生成证书

#生成私钥test2.key
openssl genrsa -out test2.key 2048
#用该私钥创建一个CSR证书签名请求文件test2.csr
openssl req -new -key test2.key -subj "/CN=test2/O=goup1/O=goup2" -out test2.csr
#用CA的私钥对该CSR请求签发一个证书test2.crt
openssl x509 -req -in test2.csr -CA /etc/kubernetes/pki/ca.crt -CAkey /etc/kubernetes/pki/ca.key -CAcreateserial -out test2.crt -days 365

设置用户

#设置用户，使用客户端的证书和客户端密钥，将证书数据嵌入到配置文件中。
kubectl config --kubeconfig=test2-config set-credentials test2 --client-certificate=./test2.crt --client-key=./test2.key --embed-certs=true
#删除用户
kubectl config --kubeconfig=test2-config unset users.test2

设置上下文和集群信息

#设置配置文件的上下文
kubectl config --kubeconfig=test2-config set-context test2@kubernetes --cluster=kubernetes --user=test2
#设置配置文件当前的上下文
kubectl config --kubeconfig=test2-config use-context test2@kubernetes
#设置配置文件的集群信息
kubectl config --kubeconfig=test2-config set-cluster kubernetes --server=https://172.16.200.70:6443/ --insecure-skip-tls-verify
#删除配置文件的集群信息
kubectl config --kubeconfig=test2-config unset clusters.kubernetes

创建ClusterRole

如下创建一个ClusterRole，对pod拥有指定的权限。

kubectl apply -f create-clusterrole.yml

create-clusterrole.yml文件如下。

kind: ClusterRole
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: test-ClusterRole
rules:
- apiGroups: [""]
  resources: ["pods"]
  verbs: ["get", "list", "watch", "create", "update", "patch", "delete"]

集群角色绑定

将刚刚创建的test-ClusterRole集群角色与test2用户进行绑定，作用在整个集群。

kubectl create clusterrolebinding test-clusterrolebinding --clusterrole=test-ClusterRole --user=test2

访问

指定刚刚生成的test2-config配置文件即可访问整个集群的pod了。

#查看上下文
kubectl config --kubeconfig=test2-config get-contexts
#访问
kubectl --kubeconfig=test2-config get pods -n kube-system

ServiceAccount

创建ServiceAccount

#在指定test命名空间下创建服务账户test3
kubectl create serviceaccount test3 -n test

创建ClusterRole

如下创建一个ClusterRole，对secrets拥有指定的权限。

kubectl apply -f create-clusterrole.yml

create-clusterrole.yml文件如下。

kind: ClusterRole
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: test-ClusterRole-secret
rules:
- apiGroups: [""]
  resources: ["secrets"]
  verbs: ["get","watch","list"]

集群角色绑定

将刚刚创建的test-ClusterRole-secret集群角色与ServiceAccount test3进行绑定，作用在整个集群。

kubectl create clusterrolebinding test3-clusterrolebinding-secret --clusterrole=test-ClusterRole-secret --serviceaccount=test:test3

访问

然后查看该test3 ServiceAccount对应的Token，用该Token访问即可。

#查看ServiceAccount对应的secret
kubectl get serviceaccounts test3 -n test -o yaml
#获取该secret的Token
kubectl describe secret test3-token-q9kf6 -n test
#用该Token访问secret
kubectl --insecure-skip-tls-verify -s https://172.16.200.70:6443 --token="xxxx" get secret
